# Spec: Storybook Stories for All Components and Pages

## Intent Description

Add Storybook stories for every application component and page, replacing the placeholder stories generated by `npx storybook init`. Each story renders the component in isolation with representative data. Components that depend on TanStack Router context (`Link`, `useRouteContext`) use a shared Storybook decorator. Pages with multiple visual states (cameras: loaded/empty/error, event detail: found/not-found, home: with/without alerts) have a story per state. Snapshot proxy images use placeholder image data. After all stories are in place, the default `src/stories/` folder and its contents are removed.

  Scenario: Placeholder stories are removed
    When browsing the Storybook sidebar

  # --- Infrastructure ---

    Given a component uses Link or useRouteContext

**Router decorator pattern**:

Components using `Link` or `useRouteContext` need TanStack Router context. The decorator creates a minimal `createRouter` + `createMemoryHistory` setup that satisfies the context requirement. The router's route tree only needs a root route — stories don't navigate.

**Mocking strategy**:

- **Header**: Pass `user` via a route context mock in the decorator. Two stories: one with a mock `SessionData`, one with `null`.
- **CamerasPage**: Prop-driven — pass different `FrigateResult<string[]>` values directly. Snapshot `<img>` tags will fail to load (404 in Storybook), which triggers the `onError` fallback showing "Snapshot unavailable". This is acceptable and actually demonstrates the fallback behavior.
- **CameraEventDetailPage**: Prop-driven — pass different `id` values. Uses `PLACEHOLDER_EVENTS` data directly.
- **CameraEventsListPage**: No mocking needed — uses `PLACEHOLDER_EVENTS` data directly.
- **HomePage**: Prop-driven — pass `error` and `status` strings.

**Constraints**:

- Stories are co-located with their components (same directory), not in a separate `src/stories/` folder
- No new dependencies needed — TanStack Router already provides `createMemoryHistory` and `createRouter`
- CSS imports: Storybook config already picks up `../src/**/*.stories.*`; `styles.css` needs to be imported in `.storybook/preview.ts` for the design system tokens to work
- `ServiceWorkerRegistration` is excluded — it renders `null` and has no visual output
