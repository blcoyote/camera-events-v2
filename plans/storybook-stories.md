# Plan: Storybook Stories for All Components and Pages

**Created**: 2026-04-14
**Branch**: main
**Status**: implemented

## Goal

Add Storybook stories for every application component and page, replacing the placeholder stories generated by `npx storybook init`. Each story renders the component in isolation with representative data. Components depending on TanStack Router context use a shared decorator. Pages with multiple visual states have a story per state. Dark mode and responsive viewports are supported via global Storybook configuration. After all stories are in place, the default `src/stories/` folder is removed.

## Risks & Open Questions

- **TanStack Router decorator**: The decorator uses `createRootRouteWithContext<{ user: SessionData | null }>()` to create a root route, then `createRouter({ routeTree: rootRoute, history: createMemoryHistory() })` wrapped in `RouterProvider`. The root route's `beforeLoad` returns the `routeContext` passed by the story. If TanStack Router's internal validation rejects this minimal route tree, the concrete fallback is to mock `useRouteContext` via `vi.mock` at the module level in each story file that needs it.
- **Storybook vitest addon**: New stories are automatically picked up by the storybook vitest project. Each story is tested for render-without-errors by default. Story render failures will break the storybook test suite, which is the desired behavior.
- **CSS import side effects**: Importing `styles.css` in preview.ts brings Tailwind and custom properties into Storybook. The styles use scoped selectors and CSS custom properties, so Storybook's own UI should not be affected.
- **Dark mode toolbar**: Uses a simple class-toggle approach (adding/removing `.dark` class on the preview `<html>` element) rather than a full Storybook theming addon, since the app already uses CSS class-based dark mode.
